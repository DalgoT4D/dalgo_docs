# KPIs

**Track your organization's Key Performance Indicators with visual progress cards, RAG status monitoring, and detailed time-series analysis.**

KPIs (Key Performance Indicators) help you monitor the most important metrics for your organization's success. Unlike charts that display raw data, KPIs define targets, track progress with visual RAG (Red-Amber-Green) status indicators, and provide contextual annotations to explain changes over time.

## What are KPIs?

A KPI in Dalgo connects a metric (a calculation from your data) to organizational targets and progress tracking:

- **Metric**: The underlying data calculation (e.g., "Number of beneficiaries served")
- **Target**: The goal value you want to achieve
- **Direction**: Whether higher or lower values are better
- **RAG Status**: Visual indicator showing if you're on track (Green), need attention (Amber), or are off track (Red)
- **Time Tracking**: Monitor progress over time periods (daily, weekly, monthly, quarterly, yearly)

## Key features

### RAG Status Monitoring
Each KPI shows a color-coded status based on your performance against targets:
- **Green**: On track (above green threshold)
- **Amber**: Needs attention (between amber and green thresholds)
- **Red**: Off track (below amber threshold)

### Flexible Time Periods
Track KPIs across different time grains:
- Daily, weekly, monthly, quarterly, or yearly
- Historical trend analysis
- Period-over-period change calculations

### Program Organization
Tag KPIs by program areas and metric types:
- **Program tags**: Group KPIs by organizational programs
- **Metric types**: Categorize as Input, Output, Outcome, or Impact metrics

### Annotations and Context
Add contextual information to explain KPI changes:
- **Notes**: Explain unusual changes or provide context
- **Beneficiary quotes**: Include direct feedback from program participants

:::info
KPIs require metrics to be defined first. You can create metrics in the [Metrics](../metrics/index.md) section, then reference them when creating KPIs.
:::
