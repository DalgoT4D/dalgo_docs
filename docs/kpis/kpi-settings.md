# KPI Settings

**Manage, edit, and organize your KPIs with display order customization, threshold adjustments, and deletion options.**

The KPI settings allow you to modify existing KPIs, reorganize their display order, and manage the overall structure of your performance monitoring system.

## Editing KPIs

### Modifying KPI configuration

1. **Open the KPI detail view** by selecting a KPI card from the dashboard.

2. **Select the edit icon** or "Edit KPI" option.

3. **Update any of the following**:
   - **Name**: Change the display name
   - **Target value**: Adjust performance targets
   - **Direction**: Switch between "higher is better" and "lower is better"
   - **Thresholds**: Modify green and amber threshold percentages
   - **Time grain**: Change tracking frequency
   - **Time dimension**: Select different date column
   - **Metric type**: Update Input/Output/Outcome/Impact classification
   - **Program tags**: Add or remove program area tags

4. **Select "Save Changes"** to update the KPI.

:::info
Changing KPI settings recalculates RAG status and may affect historical trend analysis. Consider the impact on reports and dashboards using this KPI.
:::

### When to edit KPIs

**Target adjustments**:
- Targets were set too high or low initially
- Organization capacity or context has changed
- Historical performance shows targets need recalibration

**Threshold modifications**:
- Current thresholds don't reflect meaningful performance levels
- Stakeholders want different sensitivity to performance changes
- External standards or benchmarks have changed

**Time grain changes**:
- Reporting frequency requirements have changed
- Different time periods provide better insights
- Data availability has improved or changed

## Display order management

### Reordering KPIs

1. **On the main KPIs page**, KPIs display in their configured order.

2. **Use the drag handles** (if available) to reorder KPI cards.

3. **Or edit individual KPIs** to set specific display order numbers.

### Strategic ordering principles

**Priority-based ordering**:
- Most critical KPIs first
- Strategic objectives before operational metrics
- Board-level KPIs at the top

**Program-based grouping**:
- Group related KPIs by program area
- Order programs by organizational priority
- Use consistent naming for easy scanning

**Metric type organization**:
- Impact metrics first (long-term outcomes)
- Outcome metrics second (short-term results)
- Output metrics third (direct products)
- Input metrics last (resources invested)

## Deleting KPIs

### When to delete KPIs

- **Program discontinued**: No longer relevant to organization
- **Replaced by better metrics**: New KPI provides superior insights
- **Data no longer available**: Underlying data source removed
- **Consolidation**: Merging multiple KPIs into one

### Deletion process

1. **Open the KPI detail view** for the KPI to remove.

2. **Select "Delete KPI"** or the delete option.

3. **Confirm deletion** in the dialog box.

:::warning
Deleting a KPI is permanent and removes all historical data, annotations, and trend analysis. This cannot be undone. Consider editing the KPI to mark it as inactive instead of deleting it entirely.
:::

### Before deleting

- **Export data** if you need historical information
- **Document decision** in remaining KPIs or organizational records
- **Check dependencies** - are any reports or dashboards using this KPI?
- **Inform stakeholders** who regularly review this KPI

## Bulk management

### Managing multiple KPIs

**Consistent tagging**:
- Apply program tags consistently across related KPIs
- Use standard naming conventions for easy filtering
- Update tags when program structure changes

**Threshold standardization**:
- Use similar thresholds for comparable KPIs
- Document threshold rationale for consistency
- Review thresholds annually for alignment

**Time grain alignment**:
- Use consistent time grains within program areas
- Align with reporting cycles and meeting schedules
- Consider data refresh frequency

## KPI performance optimization

### Regular maintenance

**Monthly reviews**:
- Check KPI relevance and accuracy
- Update targets based on performance trends
- Add annotations for significant changes

**Quarterly assessments**:
- Review threshold effectiveness
- Analyze KPI utility for decision-making
- Consider consolidation or creation of new KPIs

**Annual evaluation**:
- Comprehensive review of all KPI configurations
- Alignment with organizational strategy changes
- Benchmark thresholds against sector standards

### Quality assurance

**Data validation**:
- Ensure underlying metrics are calculating correctly
- Verify time dimension accuracy
- Check for data completeness and quality

**User feedback**:
- Collect input from KPI users on utility and clarity
- Adjust based on decision-making needs
- Simplify complex KPIs where possible

**Documentation**:
- Maintain rationale for each KPI's configuration
- Document threshold-setting methodology
- Keep change logs for audit and learning purposes
